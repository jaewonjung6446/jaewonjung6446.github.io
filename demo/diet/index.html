Problem to solve

해결하려는 병목

Unity Editor 의존

기획 데이터 작성과 에셋 배치가 Unity Editor 사용 가능 인원에게 집중되는 문제를 외부 편집 가능한 계약으로 분리합니다.

데이터 기준 불일치

오래된 snapshot 기준으로 만든 변경이 최신 프로젝트 상태를 덮어쓰지 않도록 stale snapshot 검증을 둡니다.

기존 런타임 경로 유지

DIET 편집 구조와 기존 XML 로딩 구조 사이에 adapter를 두어 기존 런타임 파서를 무리하게 갈아엎지 않습니다.

Workflow

공개 샘플 저장소 구성

Export

Unity 내부 XML/C# 데이터를 schema, relations, snapshot으로 분리해 외부 편집기가 이해할 수 있게 만듭니다.

Validate

외부 변경 요청을 바로 적용하지 않고, baseSnapshotHash와 현재 snapshotHash를 비교해 dry-run 리포트를 만듭니다.

Apply / Report

검증을 통과한 변경만 실제 데이터에 반영하고, 적용 결과를 작업자가 확인 가능한 report로 남깁니다.
