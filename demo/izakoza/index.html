Representative project

이자코자 · PlayerData 중심 게임플레이 시스템

상용 예정 Unity/Spine 기반 로그라이트 덱빌딩 프로젝트에서 세이브/로드, 상점·인벤토리, 스토리·이벤트 인카운터, 보상 처리, 전투 진입 연결부를 담당했습니다. 핵심은 여러 기능이 같은 플레이어 진행 상태를 기준으로 안전하게 확정되도록 만든 점입니다.

Technical value

기술적으로 보여줄 포인트

1. 진행 상태의 단일 기준

XML/JSON 정적 데이터와 PlayerData 저장 데이터를 분리하고, 상점·인카운터·보상·전투 진입 정보가 같은 PlayerData 흐름을 바라보도록 구성했습니다.

2. 저장 실패 대비

구매·보상처럼 상태가 바뀌는 기능은 변경 전 snapshot을 만들고, 저장 실패 시 런타임 상태를 복구하는 흐름으로 설명합니다.

3. 전투 진입 연결부

전투가 필요한 경우 pending combat 정보를 저장하고, 씬 전환 뒤 전투 시작 컨텍스트를 복원하는 연결부를 강조합니다.

Sample plan

공개 샘플 저장소에 넣을 것

PlayerData transaction

RunTransaction, CreateSnapshot, RestoreSnapshot, SaveWithResult를 작은 콘솔/Unity 샘플로 분리합니다.

Shop purchase rollback

결제 아이템 제거, 구매 아이템 지급, 저장 확정 중 하나라도 실패하면 구매 전 상태로 되돌리는 테스트를 포함합니다.

Encounter reward completion

보상 지급과 완료 이력이 분리되어 중복 보상이 생기는 케이스를 방지하는 흐름을 보여줍니다.
